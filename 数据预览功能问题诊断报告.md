# 数据预览功能问题诊断报告

**问题发生时间**: 2026-03-16  
**问题模块**: 数据导入中心 - 预览数据功能  
**严重程度**: 🔴 严重  

---

## 1. 问题描述

用户在点击"预览数据"按钮时，数据预览功能失败，无法显示预览内容。

### 1.1 症状表现
- 点击"预览数据"按钮后无响应
- 前端控制台显示网络错误
- API请求失败

---

## 2. 问题诊断

### 2.1 前端日志分析

从前端终端日志发现关键错误信息：

```
[vite] http proxy error: /api/data_import/upload
Error: connect ECONNREFUSED 127.0.0.1:5001
```

**关键发现**:
- ❌ 前端代理尝试连接到端口 `5001`
- ❌ 连接被拒绝 (ECONNREFUSED)

### 2.2 配置文件检查

#### vite.config.js (前端代理配置)

```javascript
server: {
  port: 5174,
  proxy: {
    '/api': {
      target: 'http://127.0.0.1:5001',  // 问题：端口5001
      changeOrigin: true
    }
  }
}
```

#### app.py (后端服务端口)

```python
app.run(debug=False, host='127.0.0.1', port=5001)  # 配置显示5001
```

#### run.py (实际运行端口)

```python
app.run(debug=True, host='0.0.0.0', port=5002)  // 实际运行在5002！
```

### 2.3 服务状态验证

| 服务 | 配置端口 | 实际运行端口 | 状态 |
|------|---------|-------------|------|
| 后端服务 | 5001 | **5002** | ✅ 运行中 |
| 前端代理目标 | 5001 | - | ❌ 配置错误 |

### 2.4 根本原因定位

**问题根源**: 端口配置不一致

1. `app.py` 配置后端运行在 5001 端口
2. `vite.config.js` 前端代理目标配置为 5001 端口
3. **但实际启动脚本 `run.py` 运行后端在 5002 端口**
4. 导致前端代理无法连接到后端服务

## 4. 解决方案

### 4.1 方案一：修改前端代理配置（推荐）

**优点**: 不需要重启后端服务  
**实施难度**: 简单  

#### 修改步骤

1. 修改 `vite.config.js` 文件
2. 将代理目标端口从 5001 改为 5002
3. 重启前端开发服务器

#### 修改内容

```javascript
// 修改前
target: 'http://127.0.0.1:5001'

// 修改后
target: 'http://127.0.0.1:5002'
```

### 4.2 方案二：修改后端运行端口

**优点**: 与app.py配置保持一致  
**缺点**: 需要重启后端服务  

#### 修改步骤

1. 修改 `run.py` 文件
2. 将端口从 5002 改为 5001
3. 重启后端服务

### 4.3 推荐方案

**推荐使用方案一**，因为：
- 后端服务已经在5002端口正常运行
- 只需要修改前端配置，影响范围小
- 实施更快速
